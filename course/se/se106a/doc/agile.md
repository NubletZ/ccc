# Agile 敏捷軟體開發

Agile Development ~= Test Driven Development (TDD)

## 觀察 1

Bug 越晚發現代價愈高，分析錯誤代價最高 (幾何成長)。

設計修改 change 代價愈前期愈高，分析錯誤的代價比測試錯誤更高。

結論：提早測試，提早發現錯誤，小循環交付，避免做出錯誤決定的代價因時間而攀高！

## 觀察 2

技術演變有利於改變 change，因素如下：

1. 打孔卡 => 雲端
2. 機器龜速 => 火箭速度
3. 中央大機房 => 個人電腦
4. 工具原始 => 工具高級且眾多
